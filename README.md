# cursor-starter

Стартер для новых репозиториев под **Cursor**: заранее задан **AI-слой** — оркестратор маршрутизации, правила экономии контекста и заготовки для bootstrap под ваш стек.

## Для чего это

- **Корневой [`CURSOR.md`](CURSOR.md)** — единая точка входа: куда смотреть агенту, какие `@`-файлы подключать, таблица маршрутизации задач.
- **[`ai-layer-builder/`](ai-layer-builder/)** — спецификации и сценарий полной настройки или обновления слоя под конкретный проект.
- **[`.cursor/rules/`](.cursor/rules/)** — постоянные политики (язык, дифф, токены, архитектура, стек).

Цель: меньше хаоса в чатах, предсказуемое поведение агента и меньше расхода контекста за счёт точечных `@` и `.cursorignore`.

## Быстрый старт

1. Скопируйте репозиторий или используйте как шаблон для нового проекта.
2. Откройте папку в **Cursor**.
3. В новом чате начните с **`@CURSOR.md`** — там карта репозитория и маршрутизация по типовым задачам.

Перед первой серьёзной настройкой под свой код откройте чат с **`@ai-layer-builder/ai-builder.md`** и пройдите bootstrap (merge с существующим `.cursor/`, не «выжигая» лишнее без запроса).

## Структура репозитория

| Путь | Назначение |
|------|------------|
| [`CURSOR.md`](CURSOR.md) | Оркестратор: маршрутизация, ссылки на rules/commands/skills |
| [`ai-layer-builder/ai-builder.md`](ai-layer-builder/ai-builder.md) | Bootstrap и обновление AI-слоя |
| [`ai-layer-builder/spec-CURSOR.md`](ai-layer-builder/spec-CURSOR.md) | Спецификация структуры `CURSOR.md` |
| [`ai-layer-builder/spec-commands.md`](ai-layer-builder/spec-commands.md) | Как описывать проектные slash-команды (не `/create-*`) |
| [`.cursor/rules/`](.cursor/rules/) | Правила Cursor (`*.mdc`) |
| [`.cursorignore`](.cursorignore) | Исключения из индексации (артефакты сборки, секреты, тяжёлые файлы) |

Проектные команды и skills появляются в `.cursor/commands/` и `.cursor/skills/` после bootstrap или по мере добавления — см. таблицу в `CURSOR.md`.

## Важные различия

- **Встроенные команды Cursor** (`/create-rule`, `/create-hook`, `/create-skill`) создают артефакты в пользовательском/проектном `.cursor/` — ими заводят правила, хуки и skills.
- **Проектные slash-команды** — это отдельные сценарии в репозитории; формат и смысл описаны в `spec-commands.md`, их не путать с `/create-*`.

## Обновление слоя

После смены стека, крупного рефакторинга или «разъезда» конфигурации снова откройте **`@ai-layer-builder/ai-builder.md`** и синхронизируйте слой; в шапке `CURSOR.md` обновите дату последнего обновления AI-слоя.

## Лицензия и вклад

Репозиторий-шаблон: при форке добавьте свой `LICENSE` и правила вклада, если планируете открытую разработку.
