# Спецификация корневого `ai-layer.md`

Файл в **корне репозитория**: `ai-layer.md`. Это **оркестратор**: куда смотреть агенту и человеку перед задачей. Не дублировать длинные правила — только ссылки и таблицы.

## Обязательные секции

### 1. Метаданные

- Одна строка: название/назначение проекта.
- `Последнее обновление AI-слоя: YYYY-MM-DD` (проставлять при каждом прогоне bootstrap).

### 2. Как начинать любую задачу в Cursor

- Сначала прочитать этот файл (или релевантный раздел).
- Подключать контекст точечно: `@путь/к/файлу`, не весь репозиторий без нужды.
- Для типовых действий: таблица «задача → команда или skill».
- Кратко **цикл принятия решения агентом**: классификация задачи по таблице маршрутизации → открыть нужные `@` (код, правила, реестр скилов) → при сомнении эскалация (план, сильная модель, вопрос человеку) — без противоречия политикам из `.cursor/rules/`.

### 3. Карта репозитория (кратко)

- 5–15 строк: корневые пакеты, где API/UI/домен, где тесты, где конфиги.
- Если monorepo — подпакеты одной строкой каждый.

### 4. Маршрутизация

Таблица в markdown, например:

| Задача | Действие |
|--------|----------|
| Новое правило | `/create-rule` (встроенная команда Cursor, не файл в репо) |
| Новый хук | `/create-hook` |
| Новый skill | `/create-skill` |
| … | … |

### 5. Ссылки на AI-слой

- `@.cursor/rules/` — перечисли **имена** ключевых `.mdc` и одну фразу зачем каждый.
- Если после bootstrap создан **реестр глобальных скилов** (напр. `global-skills-routing.mdc` / `project-skills-registry.mdc`) — **обязательно** одна строка: это **источник истины**, какие глобальные `SKILL.md` читать в каких сценариях; не дублируй полный список скилов в `ai-layer.md`, только ссылка `@…`.
- `@.cursor/commands/` — только **проектные** slash-команды (`*.md`); глобальные `/create-rule` и т.п. в репозиторий не кладут.
- `@.cursor/skills/` — перечень папок skills и когда вызывать.

### 6. Политика токенов (один абзац)

- Ссылка: `@.cursor/rules/token-economy.mdc`.
- Одно предложение: быстрая модель по умолчанию, сильная — по эскалации.

### 7. Обновление слоя

- Когда перезапускать bootstrap: смена стека, крупный рефакторинг, новая команда, накопленный хаос в `.cursor/`.
- Команда пользователю: снова открыть чат с `@ai-layer-builder/ai-builder.md`.

### 8. Опционально

- `AGENTS.md`: если используется в команде — одна строка «не дублировать длинные списки из ai-layer».

## Антипаттерны

- Вставлять полные тексты правил из `.mdc` в `ai-layer.md`.
- Описывать весь стек на несколько экранов — только границы и ссылки.
