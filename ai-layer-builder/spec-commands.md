# Slash-команды кроме create-*

Файлы в `.cursor/commands/*.md` — текст, который подставляется в чат при выборе `/имя-файла`.

## Именование

- `kebab-case.md` → `/kebab-case`.
- Имя отражает **одно** действие.

## Структура одного файла команды

1. **Цель** (1–2 предложения).
2. **Входы**: что пользователь должен указать (ветка, issue, скрин).
3. **Шаги для агента**: нумерованный короткий чеклист. **Обязательно**: первым шагом всегда предпиши составить План (например, в `@PLAN.md`).
4. **Ограничения**: что не трогать, лимиты дифа, тесты обязательны/нет.
5. **Готово**: критерий завершения. **Обязательно**: укажи требование обновить Живую Документацию (`PROJECT_KNOWLEDGE.md`) в соответствии с внесенными изменениями (без добавления логов).

## Примеры имён под типовой проект

| Файл | Назначение |
|------|------------|
| `feature.md` | Новая фича от ТЗ до PR |
| `fix-bug.md` | Воспроизведение → фикс → тест |
| `review.md` | Ревью дифа по чеклисту |
| `refactor.md` | Безопасный рефакторинг с тестами |

Создавай только те команды, которые реально экономят время в **этом** репозитории; не копируй десятки шаблонов без нужды.
