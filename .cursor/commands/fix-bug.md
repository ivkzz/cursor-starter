# /fix-bug — Воспроизведение → фикс → тест → док

Файл в `.cursor/commands/fix-bug.md` — текст, который подставляется в чат при выборе `/fix-bug`.

## Цель
Надёжно воспроизвести баг, починить его, добавить регрессионный тест, обновить Живую Документацию. Минимизировать риск регрессии.

## Входы
- Описание бага (или ссылка на issue).
- Шаги воспроизведения (если известны).
- (Опционально) Скриншот, логи, ветка, commit, где проявляется.

## Шаги для агента (обязательный порядок)

1. **Составить План** (создать/обновить `@PLAN.md` или `.cursor/plans/fix-xxx.md`):
   - Как воспроизвести.
   - Гипотеза причины.
   - Затронутые файлы.
   - Как будет выглядеть тест.
   - Что обновить в Живой Документации.

2. **Воспроизвести баг**:
   - Запустить код/тесты с шагами из входа.
   - Зафиксировать ошибку (скриншот/лог/сообщение).
   - Если не воспроизводится — уточнить у человека.

3. **Найти корневую причину**:
   - Изучить код по плану (точечный `@`).
   - Проверить связанные тесты и документацию.
   - Сформулировать минимальное исправление.

4. **Починить**:
   - Внести минимально достаточное изменение.
   - Не трогать соседние модули без необходимости.

5. **Добавить/обновить тест**:
   - Регрессионный тест, который ловит баг.
   - Убедиться, что тест был красным до фикса и стал зелёным после.

6. **Обновить Живую Документацию** (структурно):
   - Добавить в "Известные проблемы" (если не полностью починили) или удалить.
   - Обновить "Архитектура" / "Правила валидации", если баг был вызван неправильным пониманием.
   - Зафиксировать, как теперь работает исправленный поток.

7. **Проверить**:
   - Все тесты зелёные.
   - Нет регрессий в связанных фичах.
   - Готово к PR.

## Ограничения
- Не "закрашивать" симптом — искать и фиксить корневую причину.
- Лимит дифа: максимум 200-300 строк на один фикс (если больше — разбить).
- Тест обязателен.
- Не использовать сильную модель без плана на быстрой.

## Готово (критерий завершения)
- Баг воспроизведён → почищен → тест добавлен и проходит.
- Живая Документация обновлена структурно (отражает текущее состояние).
- План закрыт.
- **Обязательно**: обновлён `@PROJECT_KNOWLEDGE.md` (или эквивалент) в соответствии с изменениями (без добавления логов).
